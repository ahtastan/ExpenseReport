# Roadmap

## Phase 1, stabilize current workflow
- import current Diners statement Excel/PDF
- import receipt files
- generate authoritative statement table
- generate authoritative match table
- add review-needed queue export
- generate report workbook from statement decisions only

## Phase 2, internal MVP
- FastAPI backend
- PostgreSQL persistence
- upload UI
- review UI for uncertain matches
- statement category editing UI
- workbook + receipt pack exports

## Phase 3, coworker rollout
- user accounts
- role-based review and approvals
- multiple cardholders and periods
- team activity log
- shared templates and policy rules

## Phase 4, scale and reliability
- OCR abstraction layer
- job queue
- object storage
- reporting analytics
- audit/retention controls
